\section{AutenticaĂ§ĂŁo e AutorizaĂ§ĂŁo}
% ==============================================================================

\subsection{JWT Bearer Token}

A autenticaĂ§ĂŁo utiliza o padrĂŁo \textbf{JWT (JSON Web Token)} com algoritmo HMAC-SHA256. O fluxo ĂŠ:

\begin{enumerate}
  \item UsuĂĄrio envia email e senha para \texttt{POST /api/auth/login}
  \item O \texttt{AuthService} valida as credenciais com BCrypt
  \item Um token JWT ĂŠ gerado com as \textit{claims}: \texttt{NameIdentifier} (id), \texttt{Email} e \texttt{Name}
  \item O token ĂŠ retornado e deve ser enviado no header \texttt{Authorization: Bearer \{token\}}
  \item O middleware do ASP.NET valida o token automaticamente em todas as rotas com \texttt{[Authorize]}
\end{enumerate}

\subsection{ConfiguraĂ§ĂŁo do Token}

\begin{lstlisting}[language={[Sharp]C}]
// appsettings.json
"Jwt": {
  "Key": "chave-secreta-mĂ­nimo-32-caracteres-aqui",
  "Issuer": "AcademicEventsAPI",
  "Audience": "AcademicEventsClientes",
  "ExpiresInHours": 8
}
\end{lstlisting}

\subsection{Hash de Senha}

As senhas dos usuĂĄrios nunca sĂŁo armazenadas em texto puro. O \textbf{BCrypt} ĂŠ usado para gerar um hash com salt automĂĄtico, tornando ataques de dicionĂĄrio impraticĂĄveis mesmo que o banco seja comprometido.
